# Library Management ‚Äî Java (learning project)

A small, educational library-management application written in plain Java. It demonstrates basic object-oriented design, simple repositories, and a console UI for adding/listing/borrowing/returning books.

This repository is intended for learning and experimenting with Java: classes, collections, exceptions, and simple program flow. It is not production-ready but provides a useful starting point for building a more complete system.

## Features
- Add and remove books
- List books and search by title or id
- Borrow and return books (simple loan tracking)
- Console-based UI (`ConsoleUI`)

## Prerequisites
- Java 17+ / 21 (JDK installed and `java`/`javac` on PATH)
- A terminal and a text editor (VS Code recommended)

## Build and run
From the project root run:

```bash
# compile all sources into the bin folder
javac -d bin $(find src -name "*.java")

# run the console UI
java -cp bin main.Main
```

Notes:
- The project uses the `src` folder for sources and `bin` for compiled classes.
- If compilation fails, inspect compiler errors and fix the corresponding files in `src/`.

## Project layout

Top-level directories and important files:

- `src/` ‚Äî Java source code organized by package:
	- `main.Main` ‚Äî small launcher that starts the console UI
	- `model/` ‚Äî domain classes: `Book`, `User`, `Member`, `Librarian`, `Loan`
	- `repository/` ‚Äî in-memory repositories for `Book`, `Member`, `Loan`
	- `service/` ‚Äî `LibraryService` interface and `LibraryServiceImpl` implementation
	- `ui/ConsoleUI.java` ‚Äî console menu and user interaction
- `bin/` ‚Äî compiled class files (generated by `javac`)
- `images/` ‚Äî place diagrams or assets here (optional)

## Developer notes & suggestions
- Repositories in `src/repository` are simple in-memory lists. They must initialize their internal lists (done in current code) and return copies from `findAll()` to avoid exposing internal state.
- `model.Book` currently exposes getters/setters; prefer `private` fields and public getters/setters (this repo already follows that pattern in recent edits).
- Consider adding unit tests (JUnit) under a `test/` folder and a simple build tool (Maven/Gradle) if you plan to expand the project.
- Improve error handling: replace RuntimeExceptions with specific checked exceptions where appropriate.

## How to contribute or extend
- Add new features in `service/` and wire them to `ui/ConsoleUI.java`.
- Add persistence by replacing repository implementations with file-based or database-backed versions.
- Implement better member management (registration, authentication).

## Troubleshooting
- NPE when listing books: ensure `BookRepository` is properly initialized and `LibraryServiceImpl.getAllBooks()` returns a non-null list (it should return an empty list when no books exist).
- Compilation errors: run `javac` and fix the reported files.
